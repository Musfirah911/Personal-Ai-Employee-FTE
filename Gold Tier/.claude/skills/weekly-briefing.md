# Skill: Weekly Briefing

## Purpose
Generate the Monday Morning CEO Briefing — a comprehensive weekly audit of business performance, completed tasks, bottlenecks, and proactive suggestions. This transforms the AI Employee from reactive to proactive. The briefing is written to `/Briefings/` and the key summary is added to Dashboard.md.

## When to Use
- When a `SCHEDULED_weekly_briefing_*` file appears in `/Needs_Action/`
- Every Monday morning (triggered by scheduler.py)
- When the user explicitly asks for a weekly review or business audit
- At any point when a strategic business overview is needed

## Step-by-step Instructions

1. **Read the trigger file** — note the reporting period (start and end dates).

2. **Gather data from the vault:**

   | Source | What to extract |
   |--------|-----------------|
   | `Business_Goals.md` | Revenue target, current MTD, active projects, KPIs |
   | `/Done/` (last 7 days) | Completed tasks — filter by file modification date |
   | `/Logs/` (last 7 days) | Run `python watchers/log_summary.py --vault {vault} --days 7 --json` |
   | `/Pending_Approval/` | Overdue approvals (older than 24h) |
   | `/Plans/` | Plans in `status: in_progress` — identify stalled plans |
   | `Dashboard.md` | Last known stats for comparison |
   | `Accounting/latest_snapshot.md` | Open invoices, outstanding amount, recent payments |
   | `Social_Queue/Twitter_Queue.md` | Posts triggered/posted this week |
   | `Social_Queue/Facebook_Queue.md` | Posts triggered/posted this week |
   | `Social_Queue/Instagram_Queue.md` | Posts triggered/posted this week |

3. **Analyze the data:**

   - **Revenue:** Compare current MTD to monthly target. Calculate % achieved.
   - **Velocity:** Count tasks completed this week vs. last week (if available in logs).
   - **Bottlenecks:** Identify any plan that has been `in_progress` for >3 days without a completed checkbox.
   - **Overdue approvals:** Flag any `/Pending_Approval/` file older than 24 hours.
   - **Subscription audit:** Scan logs for recurring payments matching `SUBSCRIPTION_PATTERNS` from `Company_Handbook.md`. Flag any matching the audit rules.
   - **Proactive suggestions:** Based on goals and activity, generate 2–3 actionable suggestions.

4. **Write the CEO Briefing** to `/Briefings/{YYYY-MM-DD}_Monday_CEO_Briefing.md`:

5. **Update Dashboard.md:**
   - Update "Recent Activity" with: `{timestamp} | Weekly briefing generated | {briefing filename}`
   - Update "Quick Stats" with the week's numbers.
   - Update "Upcoming Actions" if any suggestions created Pending_Approval files.

6. **Create Pending_Approval files** for any suggestions that require action:
   - Use the `/send-email` or `/post-linkedin` skill format as appropriate.
   - Include a reference to the briefing that triggered the suggestion.

7. **Move the trigger file to `/Done/`.**

8. **Log** in today's JSONL with `action_type: weekly_briefing_generated`.

## Rules

- **Data only.** Never invent numbers. If revenue data isn't in the vault, write "Revenue data not available — check Business_Goals.md manually."
- **Be specific, not vague.** "Client B proposal took 5 days instead of 2" is useful. "Some tasks were slow" is not.
- **Always suggest 2–3 actions.** Even a great week has optimization opportunities.
- **Respect the audit rules** in `Business_Goals.md` for subscription flagging.
- **No direct actions.** The briefing is read-only. All suggested actions create Pending_Approval files — the human decides.
- **Tone:** Executive, clear, no jargon. The CEO reads this in 5 minutes over morning coffee.
